Overview

Getting Started

This guide contains real working examples, code snippets, documentation, and style guidelines.
Start by adding the following files to your page:

<link rel="stylesheet" href="assets/toolkit/styles/toolkit.css">
<script src="assets/toolkit/scripts/vendor.js"></script>
<script src="assets/toolkit/scripts/toolkit.js"></script>

Creating A New Material

Creating a new material consists of creating a new ~/src/views/*.html file and a ~/src/materials/ directory with your new material .html file in it.
For instance, say you want to create a new atom called 'My Atom':

  1. In the ~/src/views/ directory create a new file: my-atom.html with the following code: 
    
---
 fabricator: true
 title: "My Atom"
 ---

 <h1 data-f-toggle="labels" class="mb-4">\{{title}}</h1>

 \{{#each materials.my-atom.items}}

 \{{> f-item this}}

 \{{/each}}
  2. Create a new material category directory ~/src/materials/my-atom/.
    You can skip this step if your new material will be added to an existing category.
  3. Create a new .html file in your material category directory: ~/src/materials/my-atom/my-atom.html
  4. Start your new material file ~/src/materials/my-atoms/my-atom.html with the following code: 
    
---
 {
  "atomic": "atom",
  "dna": "my-atom"
 }
 ---

 <div data-dna="my-atom">The Atom Structure</div>

That seems like a lot of work, so for laziness sake; there is a built in Gulp tasks to create new materials:

gulp create:material --type "atom" --name "My Atom"
If the material already exists, you will get a notification and the task will exit.
